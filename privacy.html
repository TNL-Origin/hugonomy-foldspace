VibeAI FoldSpace – Privacy Policy

Version: 2.19.1 — Updated March 2026
Effective Date: March 2026
Developer: Hugonomy Systems
Extension Name: VibeAI FoldSpace — by Hugonomy Systems™

Summary

VibeAI FoldSpace processes all data locally in your browser. Nothing is sent to external servers. No tracking. No profiling. No telemetry. All processing begins only after you provide explicit consent.

1. Overview

VibeAI FoldSpace is a browser extension that provides real-time cognitive awareness during AI conversations. It helps you stay mentally engaged while using AI tools by detecting passive acceptance patterns and surfacing optional reflection prompts.

The extension is built on a local-first architecture: all analysis occurs on-device, and no user data is ever transmitted to external servers.

Supported platforms: ChatGPT (chat.openai.com, chatgpt.com, www.chatgpt.com), Claude (claude.ai), Gemini (gemini.google.com). VibeAI FoldSpace is not affiliated with OpenAI, Anthropic, or Google.

2. User Consent

VibeAI FoldSpace does not activate parsing, observers, or storage behavior until the user has explicitly granted consent.

All analysis is performed locally in the browser, and no conversation data is transmitted to external servers.

Before consent is given:

  • No chat content is read or parsed
  • No observers are attached to the page
  • No data is processed or stored
  • No background scanning runs

Consent is stored as a boolean flag in chrome.storage.local. You can withdraw consent at any time by clicking "Consent" in the extension footer and selecting "Revoke."

3. What Data Is Read

After consent, VibeAI FoldSpace reads the following from supported AI chat pages:

  • Visible chat messages — on-screen text in the active conversation window, read locally to detect cognitive engagement patterns (e.g., message length, tone signals)
  • Page URL — to detect which supported AI platform is active
  • DOM structure — to inject the HUD overlay interface and detect conversation elements

This reading occurs entirely within your browser. No content is transmitted externally.

4. What Data Is Stored Locally

All storage uses chrome.storage.local only — data never leaves your device.

Storage Key What It Contains User Clearable
consentGiven Boolean — whether you have consented Yes (Reset button)
consentDeferred Boolean — whether you chose "Maybe Later" Yes (Reset button)
vibeai_onboarded Boolean — whether onboarding has been shown Yes (Reset button)
vibeai_bookmarks Up to 100 user-created bookmarks of conversation moments. Contains message text snippets (max 500 chars each) saved explicitly by the user. Yes (Reset button)
vibeai_session_log Up to 50 session reflection entries (max 2000 chars each) saved explicitly by the user. Yes (Reset button)
localStorage: vibeai-hud-width HUD panel width preference (a number in pixels). UI state only — no conversation content. Yes (Reset button)
localStorage: coach panel state Coach panel and lexicon display preferences. UI state only — no conversation content. Yes (Reset button)

In addition to chrome.storage.local, the extension uses the browser's localStorage on supported AI chat pages for UI preferences only — specifically: HUD panel width, coach panel state, and lexicon display preferences. No conversation content is stored in localStorage. No sessionStorage, IndexedDB, or cookies are used.

5. What Is NOT Stored

  • Full conversation transcripts
  • Message content beyond user-initiated bookmarks
  • Personal identity or account information
  • Browsing history outside supported AI chat pages
  • Any data from pages other than chatgpt.com, claude.ai, and gemini.google.com

6. No External Transmission

VibeAI FoldSpace makes zero external network requests.

  • No data is sent to Hugonomy Systems servers
  • No analytics or crash reporting services are used
  • No advertising networks are contacted
  • No remote code is loaded or executed
  • All extension code is packaged locally within the extension itself

7. Permissions Justification

VibeAI FoldSpace requests the following permissions:

Permission Why It's Needed
storage To store consent flags, user bookmarks, and session reflections locally in chrome.storage.local
Host: chatgpt.com To inject the HUD overlay and read visible chat content on ChatGPT
Host: claude.ai To inject the HUD overlay and read visible chat content on Claude
Host: gemini.google.com To inject the HUD overlay and read visible chat content on Gemini

No other permissions are requested. The extension does not request tabs, cookies, scripting, webRequest, history, or <all_urls>.

8. User Control

You are in full control of your data at all times:

  • Revoke consent — click "Consent" in the HUD footer at any time
  • Clear all data — click "Reset" in the HUD footer to wipe all local storage
  • Disable the extension — via Chrome's Extensions page (chrome://extensions)
  • Remove the extension — uninstalling removes all associated data from your device

9. No Tracking or Profiling

VibeAI FoldSpace does NOT:

  • Track users across websites
  • Collect or infer personal identity
  • Build behavioral profiles
  • Sell, share, or transmit data to any third party
  • Use data for advertising purposes
  • Correlate your activity across sessions or devices

10. Third-Party Services

VibeAI FoldSpace does not use any third-party analytics, crash reporting, A/B testing, or advertising services. No third-party SDKs are bundled.

11. Children's Privacy

VibeAI FoldSpace does not knowingly collect data from users under 13. If you believe a child has provided data through the extension, please contact us at the address below and we will take appropriate action.

12. Changes to This Policy

If we make material changes to this privacy policy, we will update the version number and date at the top of this page. Continued use of the extension after changes constitutes acceptance of the updated policy.
